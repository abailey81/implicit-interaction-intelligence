# Release checklist

A concrete, ordered list of what must be green before tagging a new
version. Every item is mechanical enough that it can be automated in
CI ‚ÄĒ this file is the source of truth for the contract.

## 1. Code gate

- [ ] `make lint` ‚ÄĒ ruff clean, no `TODO`/`FIXME` without a dated
  follow-up issue.
- [ ] `make typecheck` ‚ÄĒ mypy clean on `i3/` and `server/`.
- [ ] `make test` ‚ÄĒ full pytest suite passes locally.
- [ ] `make test-cov` ‚ÄĒ coverage ‚Č• 80 % across `i3/` + `server/`.
- [ ] `make verify-strict` ‚ÄĒ the 44-check verification harness
  reports 0 failures under `--strict`.
- [ ] `make redteam` ‚ÄĒ all four runtime invariants pass.
- [ ] `make security-check` ‚ÄĒ bandit + pip-audit clean.
- [ ] **Iter 51 additions** (conversational coherence + intent fine-tune):
  - [ ] `python D:/tmp/context_drift_test.py` reports 170/170 (or
        equivalent on the latest expanded set).
  - [ ] `python D:/tmp/cross_session_test.py` reports 4/4.
  - [ ] If the Qwen LoRA adapter exists at
        `checkpoints/intent_lora/qwen3.5-2b_best/`, run
        `python training/eval_intent.py --backends qwen` and check
        action accuracy ‚Č• 90 %.
  - [ ] `curl http://localhost:8000/api/profiling/report` returns 200
        JSON with `fits_budget: true`.
  - [ ] `curl -X POST http://localhost:8000/api/intent -H 'Content-Type:
        application/json' -d '{"text":"set a timer for 10 minutes",
        "backend":"qwen"}'` returns
        `{"action":"set_timer","params":{"duration_seconds":600},...}`
        OR cleanly degrades to `{"error":"adapter not loaded",...}`
        when the adapter isn't trained.
  - [ ] `curl http://localhost:8000/api/intent/status` returns the
        per-backend status block.

## 2. Data gate

- [ ] `make prepare-dialogue` ‚ÄĒ pipeline runs cleanly on the bundled
  sample corpus.
- [ ] `data/corpora/sample_dialogues.jsonl` checksum matches the
  version committed at tag time.
- [ ] The quality report (`reports/.../report.json`) shows
  rejection rate < 10 % on the sample corpus.

## 3. Documentation gate

- [ ] `make docs-strict` ‚ÄĒ MkDocs Material builds with `--strict`
  (no broken links, no missing references).
- [ ] `CHANGELOG.md` has an `[X.Y.Z] ‚ÄĒ YYYY-MM-DD` section with
  non-empty `### Added` / `### Changed` / `### Fixed` / `### Security`
  as appropriate.
- [ ] `README.md` "Prerequisites" + "Installation" sections
  verified against a fresh clone.
- [ ] `CITATION.cff` version bumped and `date-released` updated.
- [ ] Every new public module has a docstring.
- [ ] Every new CLI has `--help` output that mentions its flags.

## 4. Supply-chain gate

- [ ] `.github/workflows/sbom.yml` succeeded on `main` ‚ÄĒ artefact
  attached to the previous successful run.
- [ ] `.github/workflows/scorecard.yml` score ‚Č• 7.0.
- [ ] `.github/workflows/trivy.yml` has no unacknowledged CRITICAL
  or HIGH findings.
- [ ] `.github/workflows/semgrep.yml` clean on the release SHA.
- [ ] `.github/workflows/docker.yml` produced a cosign-signed image
  with an SLSA L3 provenance attestation.

## 5. Artefact gate

- [ ] `make build` produces a wheel + sdist with no warnings.
- [ ] The wheel installs cleanly in a fresh venv (`pip install
  dist/*.whl` then `python -c "import i3; print(i3.__version__)"`).
- [ ] The Docker image built via `make docker-build` is under
  500 MB compressed.
- [ ] The edge-exported ONNX artefacts (`make export-onnx`) round-
  trip through `make verify-onnx` with parity `atol=1e-4`.

## 6. Tagging

- [ ] Bump `pyproject.toml :: [tool.poetry].version`.
- [ ] Bump `CITATION.cff :: version` and `date-released`.
- [ ] Rename the `[Unreleased]` CHANGELOG section to `[X.Y.Z] ‚ÄĒ YYYY-MM-DD`
  and add a fresh empty `[Unreleased]` above it.
- [ ] Git tag: `git tag -s vX.Y.Z -m "release X.Y.Z"` (signed).
- [ ] Push: `git push origin main vX.Y.Z`.
- [ ] GitHub release: drafted by release-please; verify the
  autogenerated notes against the CHANGELOG entry.

## 7. Post-release

- [ ] Verify published PyPI wheel signs with the release's OIDC
  identity.
- [ ] Verify the Docker image tag is immutable (pinned digest
  published alongside).
- [ ] Open the release in the Grafana dashboard for 24 hours and
  check the observability signals match the pre-release baseline.
- [ ] Close the release milestone; open the next one.
