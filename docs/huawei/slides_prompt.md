## §C — The design system (the meat)

This is **the difference between a deck that looks AI-generated and
one that looks like the candidate cared**.  Read it twice.  Then read
it once more.

### C.1 Aesthetic touchstones

These are the references the deck should evoke.  Visit each one
mentally before you start designing.

| Reference | What to take from it |
|---|---|
| **Stripe Press** (stripe.press) | Editorial typography; restrained palette; long-form respect for the reader; the "books, not blog posts" feel. |
| **Linear** (linear.app) | Dark mode that doesn't feel oppressive; razor-sharp typography; iconography that's geometric and minimal; subtle gradient *accents*, never gradient backgrounds. |
| **Bret Victor** (worrydream.com) | Diagrams that explain rather than decorate; numbers integrated into the typography rather than boxed off; the principle that **a slide is a tool for thinking**, not a billboard. |
| **Apple Keynote (Tim Cook era)** | Single-image dramatic slides interleaved with dense data slides; Helvetica-class typography; black as a background; *empty space is content*. |
| **The Pudding / NYT Graphics** | Data visualisation discipline — axis labels with units, source citations underneath, no chartjunk, no 3D, no rainbow palettes. |
| **Vercel / GitHub Universe** | Dark mode dev-tool aesthetic; sans-serif body with monospace accents; cards with subtle borders not heavy shadows. |

**The deck must not evoke**:

- A default Marp theme.
- Generic LLM-output decks (gradient backgrounds, three-bullet
  pattern on every slide, stock icons, rainbow chart colours).
- A pitch deck for a SaaS startup (animated transitions, hero
  full-bleed photos of laptops, "5 reasons why" slides).
- An academic LaTeX Beamer deck (red headers, no whitespace, all
  text).

### C.2 Colour palette

A small, opinionated palette.  At most **two non-neutral colours**
visible on any single slide.

#### C.2.1 Neutrals (background and text)

| Token | Hex | Usage |
|---|---|---|
| `--bg` | `#0E1116` | Primary background.  Near-black with a hint of warmth.  **Not** pure `#000`. |
| `--bg-elevated` | `#171B22` | Cards, callouts, code blocks. |
| `--bg-subtle` | `#1F252E` | Hover / secondary backgrounds. |
| `--border` | `#2A3441` | Card borders, dividers.  Always 1 px, never 2 px. |
| `--text-primary` | `#E6EDF3` | Body text.  **Not** pure white — pure white on near-black creates eye strain. |
| `--text-secondary` | `#8B949E` | Captions, metadata, deprecated content. |
| `--text-muted` | `#6E7681` | Footnotes, source citations. |

**Contrast check.** `--text-primary` on `--bg` = WCAG AAA (16.4:1).
`--text-secondary` on `--bg` = WCAG AA (5.8:1) — use only for
secondary information.

#### C.2.2 Per-arm semantic colours (the cascade)

The three cascade arms each get a dedicated colour, used **only** for
that arm and **only** when the arm is the slide's subject.

| Token | Hex | Arm |
|---|---|---|
| `--arm-slm` | `#7EE787` | From-scratch SLM (mint green).  Carries the "alive, growing, ours" feel. |
| `--arm-qwen` | `#F0C870` | Qwen3-1.7B + LoRA (warm amber).  Carries the "structured, deterministic" feel. |
| `--arm-gemini` | `#A5B4FC` | Gemini cloud fallback (cool lavender).  Carries the "external, last-resort" feel. |

**Critical rule.** These three colours are **never decorative**.  They
appear only when the slide is talking about the corresponding arm.  A
slide about "the project overall" uses neutrals + at most one accent.

#### C.2.3 The single accent (for headlines + emphasis)

| Token | Hex | Usage |
|---|---|---|
| `--accent` | `#F2C25B` | Headline emphasis on cover slide; hero metric numbers; the gold-pulse motif from the actuator demo. |

This single warm accent provides one consistent thread of "warmth"
through the deck without competing with the per-arm colours.

#### C.2.4 Colours **explicitly forbidden**

- **Pure black `#000`** — too harsh on near-white text.
- **Pure white `#FFF`** — too bright on dark backgrounds.
- **Pure red `#FF0000` / pure green `#00FF00` / pure blue `#0000FF`** —
  saturated primaries scream "default theme".
- **Indigo→cyan gradients** — the Web 3 / AI startup cliché.
- **Slack-purple `#4A154B`** — too associated with branded software.
- **Twitter-blue `#1DA1F2`** — same.

### C.3 Typography

#### C.3.1 Typeface stack

```
Body, headlines:  "Inter", "Helvetica Neue", -apple-system, system-ui, sans-serif
Numerics, code:   "JetBrains Mono", "Fira Code", Menlo, Consolas, monospace
```

Inter is loaded via `@import url('https://rsms.me/inter/inter.css');`
in the Marp front-matter.  Inter has *tabular numerals* via the
`font-feature-settings: "tnum"` rule — **mandatory** on every metric
display so digits align vertically across rows.

JetBrains Mono is loaded via Google Fonts.  Code blocks and all
verbatim file-paths use it.

#### C.3.2 Type scale

A precise, opinionated scale.  Do not improvise sizes.

| Role | Size (px) | Weight | Line-height | Letter-spacing |
|---|---|---|---|---|
| Display (cover slide title) | 96 | 700 | 1.05 | −0.04em |
| Title (slide H1) | 56 | 700 | 1.1 | −0.025em |
| Hero metric (giant number) | 128 | 600 | 1.0 | −0.04em |
| Subtitle (slide H2) | 28 | 500 | 1.3 | −0.01em |
| Body | 22 | 400 | 1.5 | 0 |
| Body emphasis | 22 | 600 | 1.5 | 0 |
| Caption | 16 | 500 | 1.4 | 0.02em |
| Code (inline) | 18 | 500 | 1.4 | 0 |
| Code block | 18 | 400 | 1.55 | 0 |
| Footnote / source | 13 | 500 | 1.4 | 0.02em |

**Weights you may use**: 400, 500, 600, 700.  **Never**: 100, 200, 300
(too thin, illegible at projection size), 800, 900 (hyper-bold,
shouty).

#### C.3.3 Hierarchy rules

- **One H1 per slide.** Always at the top, always at title size.
- **At most one H2 per slide.** Used only when the slide has a
  natural sub-heading.
- **Body and lists at body size (22 px).**
- **Numerics at hero-metric size (128 px) only on the dedicated
  hero-metric layout.**

#### C.3.4 Line-length discipline

Body text wraps at **≤ 60 characters per line** (this matches
editorial typography conventions for sans-serif at 22 px).  If a line
is wrapping at > 60 chars, shrink the column width or break the
sentence.

### C.4 Spacing scale (8-pt grid)

All vertical and horizontal spacing comes from this scale.  No
"7px because it looks better" decisions.

| Token | Value | Usage |
|---|---|---|
| `--space-1` | 8px | Inline gaps. |
| `--space-2` | 16px | Inside-card padding. |
| `--space-3` | 24px | Default body paragraph spacing. |
| `--space-4` | 32px | Default vertical rhythm between elements. |
| `--space-5` | 48px | Between major sections within a slide. |
| `--space-6` | 64px | Slide outer padding. |
| `--space-7` | 96px | Cover-slide vertical breathing. |
| `--space-8` | 128px | Hero-metric layout breathing. |

**Slide outer padding is always `--space-6` (64 px).** No content
touches the slide edge.

### C.7 Data visualisation discipline

Charts on slides are appropriate **only** for the perplexity-vs-data
comparison, the encoder size reduction, and the cascade routing
distribution.  No more than three charts in the deck.

#### C.7.1 Universal chart rules

- **No chartjunk.** No 3D, no drop shadows, no gradient fills, no
  legend boxes with rounded corners.
- **No rainbow palettes.** Use neutrals + at most one arm-colour or
  the `--accent`.
- **Axis labels with units.** "Perplexity (lower is better)", not
  "PPL".
- **Source line beneath every chart.** Caption size, `--text-muted`,
  formatted as `Source: reports/slm_v2_eval.md (2026-04-28)`.

## §E — Anti-patterns explicitly banned

The deck **must not** include any of these.  If you find yourself

### E.1 Visual anti-patterns

- ❌ Three-bullet pattern on every slide.
- ❌ Emoji used as decoration (✅ ⭐ 🚀 🎯 etc.).  Emoji are banned
  from the deck except where they appear *inside* a quoted code
  string from the repo.
- ❌ Stock icons (lightbulb for "idea", rocket for "launch", brain
  for "AI").
- ❌ Gradient backgrounds.  Anywhere.
- ❌ Drop shadows on cards.
- ❌ Animated transitions between slides.
- ❌ Decorative full-bleed photos behind text.
- ❌ Rounded corners larger than 4 px (except pill badges).
- ❌ More than two non-neutral colours visible on one slide.
- ❌ Helvetica Bold + ALL CAPS on body text.
- ❌ Watermarks of any kind.

### E.2 Content anti-patterns

- ❌ "5 reasons why" framing.
- ❌ "It's like Uber for X" framing.
- ❌ Any LLM-as-noun praise of the project ("groundbreaking",
  "innovative", "cutting-edge", "state-of-the-art").
- ❌ Numbers without source provenance.
- ❌ Roadmap slide with "Q1 / Q2 / Q3 / Q4" boxes.
- ❌ Team-photos slide.  This is a solo project; don't pretend
  otherwise.
- ❌ Architecture diagrams with > 12 nodes (cognitive overload).
- ❌ Code snippets longer than 8 lines.
- ❌ Repeating the same metric on multiple slides.

### E.3 Structural anti-patterns

- ❌ "Agenda" slide reading out the section names.  The dividers do
  this implicitly.
- ❌ "Thank you" slide separate from the closing slide.
- ❌ More than 15 slides total (the closing slide doesn't count).
- ❌ Any slide with > 80 words of body text.
- ❌ Running time over 30 minutes.
